HTML

  • (DOCTYPE) tells the broswer what code we are woring in
  • (Meta) this element defines metadata such as the character set, description, keywords, author, and viewport.
  • (title) is name of the project
  • (head) this element contains information about the webpage.
  • (body) this element represents the visible content shown to the user.
  • (h1,h2,h3...) are used for header lines
  • (img) is for image
  • (Src) is for the location of the image
  • (ul) is for unordered Lists
  • (li) is for listed items
  • (oi) is for ordered items
  • (footer) is for the footer of the page

CSS

  • (margin) indicates how much space we want around the outside of an element.
  • (padding) indicates how much space we want around the content inside an element.

Git

  • (git status) checks what branch we are currently on
  • (git checkout -b (branch-name)) creates a new branch and switches to it
  • (git add -A) command to commit all files in the working branch.
  • (git commit -m "message") Use the command in your terminal to commit the changes
  • (git pull origin main) use the command in your terminal to pull in the base branch
  • (git push origin feature/"name here") pushes changes to repo

JavaScript

  • (var) adds a named container that allows us to store data in our code
  • (consol.log) Prints to the console
  • (function) a set of instructions that tells the computer how to perform a certain task
  • (for) used to to perform a task on each piece of the data, using just a few lines of code
  • (if) it will check to see if the condition that we provide is truthy
  • (else if) used to add more then one conditing and output for an if statement
